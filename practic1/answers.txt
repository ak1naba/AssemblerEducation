Основные компоненты виртуальной машины фон Неймана:
Центральный процессор (CPU): выполняет инструкции.
Память: хранит данные и инструкции.
Входные/выходные устройства: для взаимодействия с пользователем.
Шины: для передачи данных и адресов между компонентами.

Шина данных и шина адреса:
Шина данных: используется для передачи данных между компонентами компьютера (например, между процессором и памятью).
Шина адреса: передает адреса памяти, чтобы указать, где находятся данные или команды.

Архитектуры CISC и RISC:
CISC (Complex Instruction Set Computer): поддерживает множество сложных инструкций, что снижает количество команд, но увеличивает сложность процессора.
RISC (Reduced Instruction Set Computer): использует меньший набор простых инструкций, что ускоряет выполнение, но требует больше инструкций для выполнения сложных задач.
Иерархия памяти и кеш-память:
Иерархия памяти: организация разных уровней памяти с разной скоростью и стоимостью (кеш, основная память, жесткий диск) для оптимизации скорости доступа.
Кеш-память: быстрое хранилище данных, которое хранит часто используемые данные для ускорения доступа процессора.

Зачем нужен ассемблер и основные команды:
Позволяет писать программы на низком уровне, близком к машинному коду.
Основные команды: MOV (перемещение данных), ADD (сложение), SUB (вычитание), JMP (переход), CMP (сравнение), NOP (нет операции).

Регистры в NASM:
Основные регистры:
32-битные: EAX, EBX, ECX, EDX (общего назначения).
Указатели: ESP (указатель стека), EBP (указатель базы).
Индексные: ESI, EDI.
Сегментные: CS, DS, SS, ES.
Флаги: EFLAGS (хранит состояние процессора).
